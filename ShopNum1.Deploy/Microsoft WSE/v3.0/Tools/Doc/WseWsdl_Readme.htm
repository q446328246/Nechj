WSDL to SoapClient Tool (WseWsdl3.exe)

The WSDL to SoapClient tool generates proxy code for Web service clients from WSDL descriptions.  The generated proxy code contains a class that derives from Microsoft.Web.Services3.Messaging.SoapClient.  Use this generated class to send SOAP messages to SOAP receivers using the SOAP Messaging model in the Web Services Enhancements for Microsoft .NET (WSE) 3.0.

WseWsdl3 {URL | path} {path to file} [language]

 

Argument

Description

URL

The URL to a WSDL file (.wsdl) or the URL to a deployed SoapReceiver or SoapService instance.

Path

The path to a local WSDL file (.wsdl).

path to file

Specifies the file in which to save the generated proxy code. The generated proxy code is a class deriving from Microsoft.Web.Services2.Messaging.SoapClient.

language

Specifies the language to use for the generated proxy class. You can specify CS (C#; default) or VB (Visual Basic) as the language argument.

 

 

 

Remarks

A .wsdl file is an XML document written in an XML grammar called Web Services Description Language (WSDL). This file defines how an Web service behaves and instructs clients as to how to interact with the service. WSDL files are also returned automatically by deployed instances of SoapReceiver and SoapService classes. You can also retrieve a WSDL file from ASP.NET Web services.

When you use WseWsdl3.exe to create a proxy class, a single source file is created in the programming language that you specify. In the process of generating the source code for the proxy class, the tool determines the best type to use for objects specified in the service description. In some cases the tool uses a least-common-denominator approach for casting objects to a type. As a result, the generated type in the proxy class might not be what the developer wants or expects. For example, when WseWsdl3.exe encounters an ArrayList type in a service description, it creates an Object Array in the generated proxy class. To ensure correct object type casts, open the file containing the generated proxy class and change any incorrect object types to the expected object type.

Examples

The following command creates a client proxy class in the C# language for an Web service located at the specified URL. The tool saves the client proxy class in the file myProxyClass.cs.

WseWsdl3 http://hostServer/WebserviceRoot/WebServiceName.ashx myProxyClass.cs

The following command creates a client proxy class in the Microsoft Visual Basic language for an Web service located at the specified URL. The tool saves the client proxy class in the file myProxyClass.vb.

WseWsdl3 http://hostServer/WebserviceRoot/WebServiceName.ashx myProxyClass.vb VB
